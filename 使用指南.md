# AiTerm v2 使用指南

## 一、快速启动

### 1. 启动服务端

```bash
cd aiterm-v2/server
bun run dev
```

服务端将在 `http://localhost:3000` 启动，提供：
- WebSocket Terminal: `ws://localhost:3000/terminal`
- WebSocket AI Chat: `ws://localhost:3000/ai/chat`
- HTTP API: `http://localhost:3000/api/*`

### 2. 启动客户端

```bash
cd aiterm-v2/client
bun run dev
```

客户端将在 `http://localhost:5173` 启动。

### 3. 访问应用

打开浏览器访问 `http://localhost:5173`

---

## 二、前置要求

### 必需环境
- **Bun** >= 1.0.0
- **Ollama** (本地运行 AI 模型)

### 安装 Ollama 并下载模型

1. 安装 Ollama: https://ollama.ai
2. 下载推荐模型:
   ```bash
   # 推荐：CodeLlama 7B (平衡性能与质量)
   ollama pull codellama:7b-instruct
   
   # 或其他模型
   ollama pull llama3.1
   ollama pull qwen2.5
   ```

### 界面布局

```
│                    AiTerm v2                            │
├──────────────────────┬──────────────────────────────────┤
│                      │                                  │

```

### Terminal 面板

- **功能**: 完整的终端模拟器，支持 PowerShell / Bash
- **操作**: 直接输入命令执行
- **快捷键**: 
  - `Ctrl+C`: 中断当前命令
  - `Ctrl+L`: 清屏

### AI Chat 面板

- **功能**: 与本地 AI 模型对话，获取命令建议和解释
- **模型切换**: 点击顶部下拉框选择已安装的 Ollama 模型
- **消息输入**: 
  - `Enter`: 发送消息
  - `Shift+Enter`: 换行
- **代码复制**: 点击代码块右上角的 "Copy" 按钮

## 四、核心功能

### 1. AI 对话

向 AI 提问，获取命令建议：

```
用户: 如何查看当前目录下的文件？
AI: 您可以使用以下命令：
    - dir (Windows)
    - ls (Linux/Mac)
```

### 2. 工具调用确认

当 AI 尝试执行命令时，会弹出确认对话框：

```
┌─────────────────────────────────────┐
│  AI 请求执行命令                     │
│                                     │
│  命令: ls -la                       │
│                                     │
│  [批准]          [拒绝]             │
└─────────────────────────────────────┘
```

- **批准**: 执行命令
- **拒绝**: 取消执行

### 3. 安全机制

应用内置多层安全保护：

1. **命令白名单**: 60+ 安全命令可直接执行
2. **危险命令拦截**: 22 种危险模式自动阻止
   - `rm -rf /`, `format c:`, `dd if=/dev/zero` 等
3. **用户确认**: 所有工具调用需用户批准
4. **路径验证**: 防止目录遍历攻击

## 五、常用操作

### 切换 AI 模型

1. 点击 AI Chat 面板顶部的模型下拉框
2. 选择已安装的模型
3. 新消息将使用新选择的模型

### 清空对话

点击 "清空对话" 按钮，清除当前会话的所有消息。

### 调整面板大小

点击面板中间的分割线，拖动调整左右面板宽度。

## 七、故障排除

### 问题: "Ollama Disconnected"

**原因**: Ollama 服务未运行

**解决**:
```bash
# 启动 Ollama 服务
ollama serve

# 或在另一个终端运行
ollama run codellama:7b-instruct
```

### 问题: AI 响应很慢

**原因**: 模型太大或硬件资源不足

**解决**:
- 使用更小的模型 (如 `codellama:7b` 而非 `codellama:34b`)
- 使用量化模型 (如 `q4_K_M` 后缀)

### 问题: WebSocket 连接失败

**原因**: 服务端未启动或端口被占用

**解决**:
```bash
# 检查端口占用
netstat -ano | findstr :3000

# 重启服务端
cd server && bun run dev
```
